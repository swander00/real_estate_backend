## 🎯 OData Query Support

Each endpoint supports full OData 4.0 queries:

```bash
# Filtering
GET /api/reso/Property?$filter=City eq 'Toronto' and MlsStatus eq 'Active'

# Field Selection  
GET /api/reso/Property?$select=ListingKey,ListPrice,City,BedroomsAboveGrade

# Sorting
GET /api/reso/Property?$orderby=ListPrice desc&$top=50

# Pagination
GET /api/reso/Property?$skip=1000&$top=1000&$count=true

# Complex Queries
GET /api/reso/ResidentialFreehold?$filter=LotDepth gt 100 and TaxAnnualAmount lt 5000
GET /api/reso/Media?$filter=ResourceRecordKey eq 'ABC123' and MediaType eq 'Photo'
GET /api/reso/PropertyRooms?$filter=ListingKey eq 'ABC123' and RoomType eq 'Kitchen'
```

## 🚨 Important Notes

### **File Naming Convention**
- The metadata endpoint must be named `$metadata.js` (with dollar sign)
- Vercel/serverless platforms handle this correctly

### **Environment Variables Required**
All endpoints use your existing environment variables:
- `SUPABASE_URL`
- `SUPABASE_KEY` 
- No additional configuration needed

### **Error Handling**
All endpoints include:
- RESO-compliant error responses
- Proper HTTP status codes
- Detailed error messages with timestamps

### **Performance Considerations**
- Each endpoint includes pagination (`$skip`, `$top`)
- Maximum 10,000 records per request
- Default 1,000 records per request
- Efficient Supabase query optimization

### **Security**
- CORS enabled for web clients
- Input validation and sanitization
- SQL injection protection via Supabase
- Rate limiting ready (add if needed)
